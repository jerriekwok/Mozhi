## 文件格式规范

每个知识库文件遵循统一的格式规范，便于RAG系统解析和检索：

### 元信息头部
```markdown
# 标题

> **类别**: [类别名]  
> **标签**: [标签1], [标签2], ...  
> **适用场景**: [场景描述]
```

### 内容结构
- 使用Markdown标题层级（`#`、`##`、`###`）组织内容
- 使用表格呈现对比信息
- 使用代码块展示学习路径或结构图
- 关键信息使用**加粗**标注

## 使用方式

### RAG接入
将本目录下的所有 `.md` 文件通过 LangChain + Chroma 进行：
1. **文档加载**：使用 MarkdownLoader 加载各文件
2. **文本分割**：按段落或标题进行分块（建议 chunk_size=500, overlap=100）
3. **向量化**：使用 embedding 模型转换为向量
4. **存储检索**：存入 Chroma 向量数据库，供 RAG 检索使用

### 3D展馆素材
文房四宝模块（文件11-14）可直接作为 Three.js 展馆中各展品的文字介绍素材。
