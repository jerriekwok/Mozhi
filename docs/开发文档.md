# 墨智（Mozhi）开发文档

## 1. 项目简介

墨智是本地运行的书法学习与创作智能助手。当前项目以“书法知识问答”为核心，已经接入本地大语言模型、向量检索知识库、学习路径推荐和 Three.js 文房四宝展馆。

当前已完成的主要能力：

- 基于 RAG 的书法问答，支持流式输出、参考资料和多轮追问。
- 基于知识库的书法学习路径推荐。
- 书法图片上传与本地视觉模型分析。
- 集字创作的前端交互基础。
- 基于 Three.js 的文房四宝三维展馆。

## 2. 技术架构

```text
浏览器（前端，8080）
  ├─ 书法问答 / 学习推荐 / 集字创作 / 文化展馆
  └─ 调用 FastAPI 接口
          │
          ▼
后端（FastAPI，8000）
  ├─ /api/chat          RAG 完整问答
  ├─ /api/chat/stream   RAG 流式问答
  ├─ /api/chat/sessions 已保存会话列表与详情
  ├─ /api/learning/plan 学习路径推荐
  ├─ /uploads/...       图片上传
  └─ /calligraphy/...   图片分析
          │
          ├─ Ollama：qwen2.5vl:3b（回答生成与图片分析）
          ├─ Ollama：bge-m3（文本向量化）
          └─ Chroma：本地向量数据库
```

## 4. 环境准备

### 4.1 必要软件

- Python 3.10 或更高版本。
- Ollama。
- Windows PowerShell 或命令提示符。

### 4.2 创建 Python 环境

在项目根目录执行：

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r backend\requirements.txt
```

### 4.3 下载模型

```powershell
ollama pull qwen2.5vl:3b
ollama pull bge-m3
```

- `qwen2.5vl:3b`：负责生成问答、学习建议和书法图片分析。
- `bge-m3`：负责将知识库文本和用户问题转成向量，用于检索。

可使用以下命令确认模型是否安装完成：

```powershell
ollama list
```

## 5. 配置说明

复制 `.env.example` 为 `.env`，常用配置如下：

```env
OLLAMA_BASE_URL=http://127.0.0.1:11434
LLM_MODEL=qwen2.5vl:3b
VISION_MODEL=qwen2.5vl:3b
MODEL_KEEP_ALIVE=-1m
EMBEDDING_MODEL=bge-m3
CHROMA_PERSIST_PATH=data/chroma_db
KNOWLEDGE_BASE_DIRS=data/knowledge,data/knowledge_base/mozhi_knowledge_base
```

`.env` 只在本机使用，不提交到 GitHub。

## 6. 知识库与 RAG

### 6.1 知识资料位置

知识库资料主要放在以下目录：

```text
data/knowledge/
data/knowledge_base/mozhi_knowledge_base/
```

目前以 Markdown 为主。新增资料后需要重建索引。

### 6.2 重建向量索引

```powershell
.\.venv\Scripts\python.exe backend\app\scripts\init_knowledge_base.py --reset
```

该命令会读取知识资料、切分内容、使用 `bge-m3` 建立向量，并写入 `data/chroma_db/`。

### 6.3 RAG 问答流程

1. 用户提问。
2. 后端将问题向量化。
3. Chroma 检索最相关的知识片段。
4. 将片段和问题一起发送给 `qwen2.5vl:3b`。
5. 返回回答与参考资料；流式接口会逐段返回文本。
6. 将问答和参考资料保存到本机 `data/mozhi.sqlite3`，刷新页面后可继续查看和追问。

## 7. 启动项目

### 7.1 一键启动

双击项目根目录的 `start_mozhi.bat`。它会打开前端和后端两个窗口：

- 前端：`http://127.0.0.1:8080`
- 后端：`http://127.0.0.1:8000`

后端接口文档地址：`http://127.0.0.1:8000/docs`。

### 7.2 手动启动

后端：

```powershell
cd backend
..\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

前端：

```powershell
.\.venv\Scripts\python.exe frontend\server.py
```

## 8. 主要接口

| 接口 | 方法 | 作用 |
|---|---|---|
| `/health` | GET | 后端健康检查 |
| `/api/chat` | POST | 返回完整 RAG 回答与资料来源 |
| `/api/chat/stream` | POST | 以 SSE 流式返回回答 |
| `/api/chat/sessions` | GET | 获取本机已保存的会话列表 |
| `/api/chat/sessions/{session_id}` | GET | 获取某条会话的完整消息与参考资料 |
| `/api/chat/sessions/{session_id}` | DELETE | 永久删除某条已保存会话 |
| `/api/learning/plan` | POST | 生成学习路径推荐 |
| `/uploads/calligraphy` | POST | 上传书法图片 |
| `/calligraphy/analyze` | POST | 调用本地视觉模型分析书法图片 |
| `/calligraphy/analyze/stream` | POST | 流式返回书法图片分析 |

详细请求和响应格式见：[前端接口说明](../frontend/api调用文档.md)。

## 9. 前端开发说明

前端不依赖 Vue、React 或构建工具，直接由 `frontend/server.py` 提供静态文件。

- 页面路由由 `frontend/js/main.js` 中的 `routeTo()` 管理。
- 新增页面时，需要同时增加 HTML 的 `data-page`、导航的 `data-route-link` 和路由白名单。
- 后端请求统一放入 `frontend/js/api.js`。
- 样式主要位于 `frontend/css/layout.css` 和 `frontend/css/components.css`。
- 展馆场景位于 `frontend/js/gallery.js`，Three.js 本地文件位于 `frontend/vendor/`，不应替换为不稳定的在线 CDN。

## 10. 测试

运行后端测试：

```powershell
.\.venv\Scripts\python.exe -m pytest backend\tests -q
```

提交前建议至少检查：

1. 后端测试通过。
2. 前端能访问首页、问答、学习推荐和文化展馆。
3. Ollama 已启动且模型可见。
4. 新增知识资料后已重建向量索引。

## 11. Git 提交规范

常用流程：

```powershell
git status
git add .
git commit -m "feat: 简要说明本次功能"
git push
```

建议的提交前缀：

- `feat:` 新功能，例如 `feat: 增加集字作品导出`
- `fix:` 修复问题，例如 `fix: 修复流式回答中断`
- `docs:` 文档更新
- `style:` 样式或界面调整
- `chore:` 配置、依赖或清理工作

以下内容已在 `.gitignore` 中排除，不应提交：

- `.env`、`.venv/`
- `data/chroma_db/`、`backend/data/chroma_db/`
- `uploads/` 中的用户图片
- `models/` 下的模型权重
- 临时文件和日志

## 12. 后续开发建议

建议按以下顺序继续：

1. 完成集字创作核心：建立合法单字素材库、自动选字、排版和图片导出。
2. 增加作品集：保存、预览和删除用户生成的作品。
3. 为文房展馆接入 RAG 问答，使用户点击器物后可以进一步提问。
4. 进一步接入专门的书法评分数据或模型，提高图片分析的专业性。
5. 增加学习打卡和长期进度记录。
