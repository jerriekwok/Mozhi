# Mozhi 项目需求说明

## 1. 项目定位

Mozhi 是一个面向书法学习与创作的智能助手平台，结合本地大模型、RAG 知识检索和 AI 图像生成能力，为用户提供知识问答、作品生成、学习辅助和数字展览能力。

## 2. 核心目标

- 书法知识问答
- AI 书法作品生成
- 书法学习辅助
- 3D 书法文化展厅

## 3. 功能模块

### 3.1 书法知识问答

- 支持用户输入问题并返回答案
- 支持上下文追问
- 支持引用本地知识库内容

### 3.2 AI 书法创作

- 支持输入关键词或主题
- 生成书法风格相关图像
- 后续可扩展到行书、楷书、草书等风格

### 3.3 书法学习推荐

- 根据用户兴趣推荐学习内容
- 支持学习顺序建议
- 支持推荐碑帖、书法家和学习路线

### 3.4 3D 文化展示

- 展示毛笔、宣纸、砚台、墨等文化元素
- 支持图文介绍和交互浏览

## 4. 技术方向

- 后端：Python + FastAPI
- 本地大模型：Ollama
- 知识检索：RAG + 向量库
- 图像生成：Stable Diffusion
- 前端：Web 页面
- 三维展示：Three.js

## 5. 数据来源

- 书法教材
- 公开资料
- 用户上传内容
- 文本、PDF、Markdown 等资料

## 6. 非功能需求

- 问答响应尽量控制在 3 到 5 秒内
- 图像生成尽量控制在 30 秒内
- 页面加载尽量控制在 3 秒内
- 注重隐私和本地部署能力

## 7. 项目阶段

1. Python 环境和项目骨架
2. FastAPI 后端与基础接口
3. RAG 知识库
4. Stable Diffusion 图像生成
5. Three.js 展示页面
6. UI 优化与联调
